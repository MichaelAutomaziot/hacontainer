# Ran Faina - Product Sync Pilot — Project Memory                                                                                                                                                                                                                                                                                             
  ## Project Context                                                                                                                                                                                                                                                                                                                            
  - **Scope:** Build a cloud-hosted central product dashboard and automation that compares The Container catalog against Super-Pharm, identifies missing products, applies the client's pricing/shipping/promo rules, and publishes the approved products through Mirakl while preparing the system for future multi-channel expansion.

                                                                                                                                                                                                                                                                                                                                                  ## Key Decisions                                                                                                                                                                                                                                                                                                                              
  - Pilot scope starts with The Container/Konesh B'O as source and Super-Pharm Marketplace as the only target channel; Zap and Walla are deferred to a later phase.
- A custom central Product Hub/dashboard will be built so future products are entered once and synced outward from one place.
- The preferred deployment model is a cloud server rather than running on the client's local computer.
- Pricing logic for Super-Pharm was defined as using the current selling price from Konesh B'O plus configured source shipping cost, with a 'before sale' price set about 15% higher, plus a default 39 ILS Super-Pharm shipping fee.
- Super-Pharm promo dates should default from upload date through one month later for each product.
- Non-essential shipping upgrades and special delivery surcharges from the source site should not be copied into Super-Pharm.
- If a product already has competing offers on Super-Pharm, matching the lowest visible offer was discussed as a desired business rule, while still keeping the default shipping treatment.
- Dashboard product fields should generally mirror the categories/filters already present in The Container catalog.
- Cloud accounts/services are intended to be opened under the client's RanFina email, coordinated primarily through Yossi.
